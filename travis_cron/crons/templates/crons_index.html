At youtube-dl we love Travis CI, the Free Hosted Continuous Integration Platform for the Open Source Community. But, as what we do is scraping, and websites change even if our code doesn't, we need to run tests periodically.

The awsome guys at Travis CI are working on implementing such a feature, but in the mean time this little web app allows you to set up daily, weekly, or personalized (feedback on better schedules appreciated) triggers for your project tests.

Activate a new trigger

At the moment new triggers are manually moderated, but I'll try to be as fast as possible. Please don't abuse this service, it is meant only for projects that need regular testing.

You will be taken to a GitHub authorization page. We'll use the GitHub token only to exchange it for a Travis CI one. As the Travis API is still new, this is still the only implemented way to authenticate through OAuth.
